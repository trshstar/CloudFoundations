Cloud Foundations: I Eat Very Crunchy Seaweeds

A web hosting architecture of core AWS services where: IAM manages service permissions, EC2 hosts the Apache web server, VPC provides network isolation and connectivity, CloudFront delivers faster content, and S3 stores assets such as images.

Project Workflow

AWS Services

IAM

  • Manages user credentials
  • Provides permissions and access to EC2, S3, CloudFront services
  • Test - Assigned to the admin group with AdministratorAccess to have control over all the services in the console and act as the root
  • J-Test - Assigned to the medium-access group with AmazonEC2InstanceConnect and AmazonEC2FullAccess to ensure the minimal privilege needed to complete the job. Ex: launch a EC2 instance, SSH connection to the EC2 instance
  • K-Test - Assigned to the least-access group with AmazonEC2ReadOnlyAccess to grant access to view the EC2 resources. The user cannot perform any actions like launch, terminate, or modify instances. No ssh connection to the instance as well.

VPC

  • It's like a house - private space
  • The public subnet is like a living room (private would be like the bedrooms)
  • I place my EC2 instance in the living room like a TV
  • My security groups are like the house rules about what can they do or who can enter
  • I connect to internet via Internet Gateway which is my front door connecting to the street
  • Route tables is like the hallway directs the living room to the front door

EC2

  • Runs the Apache server and host the HTML content
  • Located within VPC subnet
  • Accessible through VPCâ€™s networking

CloudFront

  • Content delivery network
  • Pulls contents from S3 bucket and provides fast content delivery
  • Reduces load on EC2

S3

  • Stores images and hosts the images
  • Accessible though CloudFront
  • IAM plays a role here to provide permissions to users to have access in this service
